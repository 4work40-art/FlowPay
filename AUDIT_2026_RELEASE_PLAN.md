# Release Plan — сводный релиз по итогам аудитов 2026

Утверждено: берём в работу весь бэклог из `AUDIT_2026_MASTER.md`, **кроме**:
- PAN-1 (вход по SMS) — исключено по требованию
- PAN-2 (Telegram-уведомления) — исключено по требованию, Telegram не
  добавляем нигде в продукте

Регистрация — только email (уже так и было, дополнительно подтверждено).
Добавлен юридический блок (152-ФЗ), которого не было ни в одном из
предыдущих аудитов — правовой пробел, а не пожелание пользователей.

Формат: один релиз, все задачи на одной ветке, единый прогон проверки перед
деплоем, один merge в `main`.

## Атомарные задачи по исполнителям

### Legal/Compliance
- **LEGAL-1**: согласие на обработку персональных данных при регистрации
  (152-ФЗ) — обязательный чекбокс, фиксация времени согласия, страницы
  «Политика конфиденциальности» и «Публичная оферта»

### Fintech Dev
- **GLB-1**: публичная read-only ссылка на счёт (без регистрации, без
  токена — invoice.id уже непредсказуемый UUID)
- **PAN-3**: импорт банковской выписки (CSV) — автосопоставление платежей
  по номеру счёта в назначении платежа
- **GLB-3**: автонумерация счетов (опционально, если поле не заполнено)
- **MKT-3**: приглашение коллег в организацию (email-инвайт, без Telegram)
- **PAN-6**: CSV-экспорт — фильтр по датам
- **PAN-7**: логотип организации (загрузка, отображение)
- **UX-2-доп**: PDF-квитанция об оплате

### UX/UI
- **UX-1**: дашборд — один главный показатель + сворачиваемые детали
- **UX-3**: онбординг-чеклист для новой организации
- **PAN-4**: мобильный аудит + фикс критичных проблем
- **PAN-5**: тултипы к статусам счёта

### Маркетолог
- **MKT-1/2**: публичный лендинг + публичная страница тарифов (без логина)

### Продукт/поддержка
- **SUP-1**: канал поддержки — email вместо мессенджера (Telegram исключён
  по вашему требованию), ссылка в интерфейсе

## Не берём в этот релиз (осознанно)
- PAN-8 (шаблоны повторяющихся счетов) — низкий приоритет
- Двусторонняя модель долга и мультиорганизационный доступ бухгалтеров —
  требуют отдельного решения об архитектуре, не готовы к оценке
