# Сводка проекта — Счёт&Контроль

Дата: 12 июля 2026 · Статус: боевая версия развёрнута и доступна публично

## Что это

«Счёт&Контроль» — сервис учёта дебиторской задолженности: счета, платежи, контрагенты, статусы оплат, отчётность. Изначально был MVP-прототипом с демо-данными без авторизации; за это взаимодействие доведён до рабочего состояния с реальной защитой данных и полным набором базовых сценариев.

## Что сделано

### Инфраструктура
- Развёрнут на VPS через Docker Compose (Postgres, Redis, API, фронтенд)
- Фронтенд переведён с dev-сервера на боевую сборку Next.js (`next build && next start`)
- UFW-файрвол: открыты только порты 22/3000/3001, админ-панели (Adminer, Redis Commander) закрыты от внешнего доступа
- Ежедневные автобэкапы БД по крону, хранение 14 дней
- Однострочный скрипт `deploy.sh` — установка Docker, миграции, пересборка, файрвол, бэкапы за один проход

### Безопасность
- Настоящая авторизация: подписанные JWT-токены (было — незащищённый API, читаемый и изменяемый кем угодно)
- Middleware проверки токена на всех эндпоинтах, отзыв токена через Redis при выходе
- Организация и пользователь берутся из токена, а не из тела запроса — устранена возможность подмены
- Устранена гонка при одновременных платежах (транзакция с блокировкой строки)
- Устранена утечка контрагентов между организациями
- `JWT_SECRET` обязателен — сервер не запустится со слабым значением по умолчанию
- Сырые ошибки БД не улетают клиенту как есть

### Функциональность (по итогам независимого аудита QA/UX/продукт)
- Формы создания счёта и фиксации платежа, страница деталей счёта — ранее вели в никуда, хотя backend уже был готов
- CRUD контрагентов с карточкой и подсчётом долга
- Полный жизненный цикл статуса счёта, включая ранее недостижимые «Просрочен» и «Спор» (с выходом из спора)
- Страница настроек: смена пароля, состав команды
- Поиск по счетам и контрагентам, пагинация платежей, экспорт в CSV
- Отчёт по возрасту просроченной задолженности
- Лимит счетов по тарифу организации теперь реально применяется

### Кабинет создателя
Отдельная панель (`/admin`) для владельца платформы — сводная статистика по всем организациям: рост регистраций, распределение по тарифам, активность. Доступ по флагу `is_platform_admin` в БД.

### Локализация
Интерфейс и все сообщения API переведены на русский — включая ошибки валидации, которые раньше приходили с бэкенда на английском.

## Известные ограничения (осознанно отложено)

| Ограничение | Причина |
|---|---|
| Нет HTTPS/домена | Решение пользователя — работаем по IP/HTTP на этом этапе |
| Нет самостоятельной регистрации новых организаций | Отдельная фича мультиарендности, см. ROADMAP.md |
| Нет приглашения новых пользователей в команду | Роли в БД уже готовы, не хватает только формы |
| Нет вложений файлов к счетам | Требует отдельное хранилище (S3), не входило в объём этого захода |
| Нет уведомлений о просрочке | Требует планировщик + канал доставки (email/Telegram) |
| 1 ГБ RAM на сервере | Достаточно для старта, может понадобиться апгрейд при росте нагрузки |
