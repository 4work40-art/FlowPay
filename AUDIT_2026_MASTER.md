Собирает воедино четыре документа: `AUDIT_2026.md` (мультидисциплинарный
аудит), `AUDIT_2026_USER_PANEL.md` (синтетическая панель 20 персон),
`AUDIT_2026_GLOBAL.md` (сверка с мировой UX-практикой), и ниже — новый слой:
**что реальные пользователи пишут в отзывах на аналоги** (Bill.com, Wave,
FreshBooks, QuickBooks, СБИС, Контур.Эльба — Trustpilot, Capterra, G2,
Otzovik, iRecommend). Ничего не внедряется — это вход для вашего решения.

## Новый слой: боли из отзывов на аналоги

### Универсальная боль №1 — поддержка (встречается у ВСЕХ аналогов без исключения)
Bill.com (1.3★ Trustpilot, «поддержка практически не существует»), Wave
(1.3★, «зависшие деньги, поддержка не отвечает»), СБИС («до недели ждать
ответа, и за ответы требуют денег»), Контур.Эльба («у техподдержки нет
желания решать проблемы интерфейса»). Это не совпадение — плохая поддержка
у финтех-инструментов почти неизбежна при масштабе. Для нас на текущей
стадии (единицы-десятки пользователей) это — **возможность отличаться**, а
не задача на бэклог: реальный человек, отвечающий быстро, дешевле любой
фичи.
→ **Новая задача SUP-1**: зафиксировать канал поддержки (Telegram-чат
с реальным ответом) как часть продукта, не как «когда-нибудь потом».

### Bill.com/Wave — тревожные паттерны про деньги
«Аккаунт заблокировали без предупреждения через 30 дней», «средства
зависают на дни/недели без объяснений». Для нас с ЮKassa прямого риска
нет (мы не держим деньги — это делает ЮKassa), но паттерн важен для
GLB-1 (публичная ссылка на счёт) и для коммуникации статуса подписки:
любое автоматическое действие с тарифом/лимитом должно быть заранее
объяснено, не «сюрпризом». Наша текущая реализация `/billing/cancel` с
подтверждающим диалогом уже соответствует этому принципу — отмечаю как
**уже сделано правильно**, изменений не требует.

### QuickBooks — путаница с нумерацией счетов
«Номера счетов не идут подряд, есть пропуски — путает в учёте». У нас
`number` — свободное текстовое поле, которое пользователь вводит вручную
при каждом счёте. Тот же риск: опечатался, задвоил номер, пропустил.
→ **Новая задача GLB-3**: опциональная автонумерация счетов (последовательный
номер по умолчанию, с возможностью переопределить вручную).

### QuickBooks — «всё стоит денег сверху»
Скрытые доплаты за функции — топ-жалоба на прайсинг. У нас уже сделан
`/billing/plans` с открытыми ценами (MKT-2 из первого аудита), но пока
недоступен без логина. Это усиливает, не меняет приоритет MKT-2.

### СБИС — жёсткий годовой контракт без помесячной оплаты
Прямая жалоба клиентов. У нас модель уже помесячная (ЮKassa разовые
платежи, без принудительного годового лока) — **валидирует уже принятое
архитектурное решение**, действий не требует.

### FreshBooks — то, за что хвалят (не жалоба, а сильная сторона конкурента)
Client self-service portal (пересекается с GLB-1), прикрепление расходов к
счёту, автоматические напоминания о просроченной оплате. Напоминания уже
запланированы (PAN-2). Прикрепление расходов — вне нашего фокуса
(«не бухгалтерия», по фундаменту) — сознательно не берём.
